# DevOps – List of Software & Tools

---

## 1. Docker and Docker Desktop

### Aim:
Docker and Docker Desktop

Brief:
Containerization platform to build, package, and run applications in isolated environments. Docker Desktop provides GUI and local Kubernetes support.

---

## 2. pgAdmin

### Aim:
pgAdmin

Brief:
GUI management tool for PostgreSQL databases. Used for querying, monitoring, and managing database objects.

---

## 3. MySQL

### Aim:
MySQL

Brief:
Relational database management system used for structured data storage in applications.

---

## 4. Flameshot

### Aim:
Flameshot

Brief:
Screenshot tool with annotation support, useful for documentation, debugging, and reporting issues.

---

## 5. Visual Studio Code

### Aim:
Visual Studio Code

Brief:
Lightweight but powerful source code editor with extensions for DevOps tools, containers, cloud, and remote development.

---

## 6. Notepad++

### Aim:
Notepad++

Brief:
Lightweight text editor mainly used in Windows for quick editing of configs and scripts.

---

## 7. Azure CLI

### Aim:
Azure CLI

Brief:
Command-line tool to manage Azure cloud resources programmatically.

---

## 8. AWS CLI

### Aim:
AWS CLI

Brief:
Command-line tool to manage AWS services like EC2, S3, IAM, and more.

---

## 9. Git

### Aim:
Git

Brief:
Distributed version control system used to track code changes and collaborate efficiently.

---

## 10. Nginx

### Aim:
Nginx

Brief:
High-performance web server and reverse proxy used for load balancing and serving static content.

---

## 11. Apache

### Aim:
Apache

Brief:
Popular web server used to host web applications and handle HTTP requests.

---

## 12. Zsh

### Aim:
Zsh

Brief:
Advanced shell with improved scripting, auto-completion, and plugin support over bash.

---

## 13. Helm

### Aim:
Helm

Brief:
Kubernetes package manager used to deploy and manage applications using Helm charts.

---

## 14. Python and pyenv

### Aim:
Python and pyenv

Brief:
Python is used for automation and scripting. Pyenv manages multiple Python versions on the same system.

---

## 15. Node.js

### Aim:
Nodejs

Brief:
JavaScript runtime environment used for backend services and DevOps tooling.

---

## 16. MySQL (Duplicate Entry)

### Aim:
Mysql

Brief:
Already listed above. (Can remove duplicate if not intentional.)

---

## 17. Terraform

### Aim:
Terraform

Brief:
Infrastructure as Code (IaC) tool used to provision and manage cloud infrastructure declaratively.

---

## 18. GitHub CLI

### Aim:
GitHub CLI

Brief:
Command-line tool to interact with GitHub repositories, PRs, issues, and workflows.

---

## 19. jq

### Aim:
Jq

Brief:
Lightweight JSON processor used in shell scripts to parse and manipulate JSON data.

---

## 20. htop

### Aim:
htop

Brief:
Interactive system monitoring tool to view CPU, memory, and process usage.

---

## 21. Terragrunt

### Aim:
Terragrunt

Brief:
Wrapper for Terraform that simplifies managing multiple environments and reduces code duplication.

---

## 22. NVM

### Aim:
Nvm

Brief:
Node Version Manager used to install and manage multiple Node.js versions.

---

## 23. Ngrok

### Aim:
Ngrok

Brief:
Secure tunneling tool to expose local services to the internet for testing webhooks and external integrations.
